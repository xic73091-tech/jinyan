# 诚言编程语言 — 使用说明 / Chengyan Language — User Guide

---

## 安装 / Installation

### 从源码编译 / Build from Source

```bash
# 前提：安装 Rust / Prerequisites: Install Rust (https://rustup.rs)
git clone <仓库地址>
cd 诚言编译器
cargo build --release
```

编译完成后，可执行文件位于 `target/release/chengyan`。
After building, the executable is located at `target/release/chengyan`.

### 环境变量 / Environment Variables

将 `target/release` 目录加入系统 `PATH`，即可在任意位置使用 `chengyan` 命令。
Add the `target/release` directory to your system `PATH` so you can use the `chengyan` command from anywhere.

## 语言快速入门 / Language Quick Start

### 变量 / Variables

```诚言
让 姓名 = "诚言"           // let name = "Chengyan"
让 年龄 = 1                // let age = 1
```

### 条件 / Conditionals

```诚言
如果 温度 > 100 {          // if temperature > 100
    打印("水开了")          //   print("The water is boiling")
} 否则如果 温度 > 0 {      // else if temperature > 0
    打印("液态水")          //   print("Liquid water")
} 否则 {                   // else
    打印("冰")             //   print("Ice")
}
```

### 循环 / Loops

```诚言
// 当循环 / While loop
让 计数 = 0                // let count = 0
当 计数 < 10 {             // while count < 10
    计数 = 计数 + 1        //   count = count + 1
}

// 遍历 / For-each
对 颜色 在 ["红", "绿", "蓝"] 中 {   // for color in ["red", "green", "blue"]
    打印(颜色)                        //   print(color)
}
```

### 函数 / Functions

```诚言
定义 加法(甲, 乙) {                    // fn add(a, b)
    返回 甲 + 乙                       //   return a + b
}

定义 阶乘(值) {                        // fn factorial(value)
    如果 值 <= 1 { 返回 1 }            //   if value <= 1 { return 1 }
    返回 值 * 阶乘(值 - 1)             //   return value * factorial(value - 1)
}
```

### 类 / Classes

```诚言
类 点 {                                // class Point
    构造(x, y) {                       //   constructor(x, y)
        本.x = x                       //     self.x = x
        本.y = y                       //     self.y = y
    }

    方法 距离(其他) {                   //   method distance(other)
        让 dx = 本.x - 其他.x          //     let dx = self.x - other.x
        让 dy = 本.y - 其他.y          //     let dy = self.y - other.y
        返回 dx * dx + dy * dy        //     return dx * dx + dy * dy
    }
}

让 p1 = 点(3, 4)                       // let p1 = Point(3, 4)
让 p2 = 点(0, 0)                       // let p2 = Point(0, 0)
打印(p1.距离(p2))                      // print(p1.distance(p2))
```

### 数组和映射 / Arrays and Maps

```诚言
让 数字 = [1, 2, 3, 4, 5]                                      // let numbers = [1, 2, 3, 4, 5]
让 信息 = {"姓名": "诚言", "版本": "1.0"}                      // let info = {"name": "Chengyan", "version": "1.0"}

打印(数字[0])           // 1                                    // print(numbers[0])
打印(信息["姓名"])      // 诚言                                 // print(info["name"])
打印(长度(数字))        // 5                                    // print(length(numbers))
```

## 常见问题 / FAQ

| 问题 | Question | 回答 | Answer |
|------|----------|------|--------|
| 诚言和 Python 有什么区别？ | How is Chengyan different from Python? | 诚言是编译型语言（编译为字节码后在虚拟机执行），不是解释型。诚言的编译器和虚拟机完全从零构建，不依赖任何现有运行时。 | Chengyan is a compiled language (compiled to bytecode, then executed on a VM), not interpreted. The compiler and VM are built entirely from scratch with no dependency on any existing runtime. |
| 为什么用 Rust 写编译器？ | Why use Rust to write the compiler? | Rust 内存安全、性能好、模式匹配强大，适合写编译器。诚言本身不依赖 Rust 运行时，编译器只是构建工具。 | Rust offers memory safety, high performance, and powerful pattern matching — ideal for writing compilers. Chengyan itself does not depend on the Rust runtime; the compiler is only a build tool. |
| 如何贡献？ | How can I contribute? | 欢迎提交代码、报告问题、改进文档。请使用中文交流。 | Code contributions, issue reports, and documentation improvements are welcome. Please communicate in Chinese. |
| 支持哪些平台？ | Which platforms are supported? | 支持 Windows、macOS、Linux（x86-64、ARM64）。 | Windows, macOS, Linux (x86-64, ARM64). |
| 性能如何？ | How is the performance? | 作为字节码解释型语言，性能介于 Python 和 JavaScript 之间。未来计划支持 JIT 编译以提升热点代码性能。 | As a bytecode-interpreted language, performance falls between Python and JavaScript. JIT compilation for hot code paths is planned for the future. |
